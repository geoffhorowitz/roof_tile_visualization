# Vertex AI Migration Execution Update

This document describes what was tried, what worked, and the results of the Vertex AI migration.

## Approach & Implementation

1. **Environment Variables**:
   - Updated `.env.example` and `.env.local` to include Vertex AI configuration fields (`VERTEX_USE_ENDPOINT`, `VERTEX_PROJECT_ID`, `VERTEX_LOCATION`, `VERTEX_ENDPOINT_ID`, `VERTEX_SERVICE_ACCOUNT_KEY`) with detailed instructions on how to acquire and configure them.

2. **Application Configuration**:
   - Added a `vertex_ai_settings` object to `config/advanced_user_configs.json`.
   - Updated `utils/configLoader.ts` to dynamically resolve configuration using environment variables as overrides.

3. **Vertex AI Prediction Client**:
   - Implemented `utils/vertexClient.ts` to handle OAuth2 authentication and payload dispatch.
   - For local development, the client signs a JSON Web Token (JWT) using the Node.js standard `crypto` library and exchanges it for a Google OAuth access token at `https://oauth2.googleapis.com/token`.
   - For Cloud Run production, the client fetches the access token from the GCP Metadata Server (`http://metadata.google.internal/computeMetadata/v1/instance/service-accounts/default/token`) for zero-key deployment safety.

4. **Next.js Generate API Route Routing**:
   - Modified `app/api/generate/route.ts` to check `configs.vertex_ai_settings.use_vertex_ai`.
   - When enabled, the route converts the uploaded image to a base64 string, constructs the ComfyUI workflow JSON schema (injecting the default image name `"input.png"`), calls `predictVertexAI()`, and decodes the response base64 string into a Buffer.
   - When disabled, the route executes the standard local ComfyUI queuing and polling flow.
   - All downstream Supabase Storage upload, local saving, and DB recording steps remain intact.

5. **Verification Script**:
   - Created `tests/test_vertex_ai.ts` to load the default house image, read configuration, and test connection to Vertex AI.

6. **Production Deployment Documentation**:
   - Created a comprehensive guide `docs/VERTEX_AI_SETUP.md` detailing how to containerize ComfyUI, push the image to Artifact Registry, deploy to a Vertex AI prediction endpoint with GPU acceleration, and configure Next.js environment variables.

---

## What Worked

- **Next.js Integration**: The dual-mode route is functional. If Vertex AI is disabled, it uses the local ComfyUI API. If enabled, it uses the Vertex AI Predict API.
- **Node.js OAuth Signatures**: Signing the Google Service Account JWT works using standard Node `crypto` library without adding heavy and potentially unstable native dependencies like `@google-cloud/aiplatform`.
- **Build Compilation**: Running `npm run build` compiled the entire Next.js application without any TypeScript or routing errors.
- **Verification Script**: Running `npx tsx tests/test_vertex_ai.ts` successfully verified that the script correctly imports the configs, handles base64 image parsing, and reports configuration warnings gracefully.

---

## What Didn't Work / Adjustments

- **TypeScript JSON Import Type Inference**: When adding `vertex_ai_settings` to `config/advanced_user_configs.json`, TypeScript didn't immediately infer the new fields on the imported JSON object in `utils/configLoader.ts`. This was resolved by casting `rawConfigs` to `any` when reading `vertex_ai_settings`.
