# Align Checkpoint Configuration to SDXL Base

Testing and evaluation in `agent_artifacts/2026.06.09_update_inpainting_checkpoint_v2.md` established that the standard base SDXL checkpoint (`sd_xl_base_1.0.safetensors`) is the correct model for the generation workflow due to severe color prompt adherence bugs in the custom inpainting model. 

However, the Dockerfile created during Vertex AI migration (`docker/Dockerfile`) mistakenly downloads the base model but saves it as the inpainting model filename. Additionally, some older test suites still hardcode the custom inpainting model filename.

This plan details the steps to correct the Dockerfile download name to `sd_xl_base_1.0.safetensors` and update the remaining test files.

## Proposed Changes

### Docker Configuration
#### [MODIFY] [Dockerfile](file:///home/gh-wsl/github_repos/personal/roof_tile_visualizer/docker/Dockerfile)
* Correct the `-O` destination filename in the `wget` command to save the model under its correct name (`sd_xl_base_1.0.safetensors`) instead of the custom inpainting name:
  ```dockerfile
  RUN wget -O models/checkpoints/sd_xl_base_1.0.safetensors https://huggingface.co/stabilityai/stable-diffusion-xl-base-1.0/resolve/main/sd_xl_base_1.0.safetensors
  ```
