# ComfyUI API Integration Design Document

This document outlines the implementation plan for connecting the Next.js frontend to your local ComfyUI API. Now that your RTX 3060 local server is reachable, we will build the bridge to send user uploads to the AI and retrieve the generated images.

## User Review Required
> [!IMPORTANT]
> Please review the ComfyUI integration workflow below. The ComfyUI API is asynchronous, meaning we have to upload the image, queue the prompt, and then poll for the result.

## Open Questions
> [!NOTE]
> **Next.js vs Streamlit/Gradio**: You asked if Next.js is better than Streamlit or Gradio. Streamlit/Gradio are excellent and much faster for prototyping Python AI scripts. However, Next.js was chosen here because you requested a highly customized, premium web application with glassmorphism, precise animations, and a roadmap to include user authentication (Supabase). Gradio/Streamlit are extremely rigid visually; achieving the "WOW" factor you requested is very difficult with them, whereas Next.js gives us 100% control over the product's UX and aesthetics.
>
> **Workflow JSON Strategy**: You asked why we are using a placeholder workflow instead of building it together. We *will* build it together! However, ComfyUI workflows (the exact combination of nodes, model names, ControlNet strengths, and denoising values) are highly dependent on the specific models you downloaded to your RTX 3060. 
> 
> The best strategy is:
> 1. We wire up the Next.js API using a simplified "placeholder" JSON just to prove the frontend can talk to your backend (send image -> process -> receive image).
> 2. Then, you open the ComfyUI visual interface in your browser, and we work together to build the perfect roof-replacement node graph. 
> 3. Once it looks perfect in the visual interface, you click "Save (API format)" to export the JSON, and we drop it into our Next.js code. This is the standard developer workflow for ComfyUI.

## Proposed Architecture

### ComfyUI API Communication Flow
We will create a Next.js Serverless Function (`app/api/generate/route.ts`) that handles the following sequence securely on the backend:

1. **Upload Original Image**: The Next.js API receives the multipart form data (image + selected style prompt) from the frontend. It then forwards the image via `POST` to ComfyUI's `/upload/image` endpoint to place it in ComfyUI's `input` directory.
2. **Submit Workflow**: The API constructs the ComfyUI Workflow JSON. It injects the filename of the uploaded image into the `LoadImage` node and the user's selected style into the `CLIPTextEncode` (Prompt) node. It sends this JSON to the `/prompt` endpoint.
3. **Poll for Completion**: The API polls ComfyUI's `/history/{prompt_id}` endpoint every 2 seconds. ComfyUI processes the image on your RTX 3060.
4. **Retrieve Result**: Once the history indicates completion, the API extracts the output filename, fetches the final generated image bytes from ComfyUI's `/view` endpoint, and returns it to the Next.js frontend.
