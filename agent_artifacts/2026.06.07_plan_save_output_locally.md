# Plan: Save Generated Output Locally as Interim Step

This plan details how to implement saving the generated roof visualizer output to the local `data/output/` folder as an interim step before Supabase cloud storage integration is implemented, and updating `TODO.md` accordingly.

## Proposed Changes

### Next.js Backend API

#### [MODIFY] [route.ts (generate)](file:///home/gh-wsl/github_repos/personal/roof_tile_visualizer/app/api/generate/route.ts)
- Modify `/api/generate` to download the generated image from ComfyUI after the generation completes.
- Save the downloaded image to `data/output/<filename>` in the project workspace.
- Handle directory creation for `data/output` if it does not exist.

#### [MODIFY] [route.ts (image)](file:///home/gh-wsl/github_repos/personal/roof_tile_visualizer/app/api/image/route.ts)
- Modify `/api/image` (the image proxy endpoint) to first check if the requested file exists in `data/output/`.
- If the file exists locally, serve it directly from the local file system.
- If not, fall back to fetching it from ComfyUI, saving it to `data/output/`, and serving it.
