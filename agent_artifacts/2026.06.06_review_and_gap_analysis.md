## 2. Key Technical Gaps Explained

### Gap 1: Inpainting vs. Whole-Image Generation (ComfyUI Workflow)
* **The Problem:** The current ComfyUI workflow queued in `app/api/generate/route.ts` is a standard Image-to-Image pipeline. Because the `denoise` is set to `0.85` and there is no mask, the AI attempts to redraw the *entire* image. This means the house structure, landscaping, windows, and sky will shift and change along with the roof.
* **The Solution:** We must implement a segmentation and masking step.
  1. **Option A (AI-driven):** Integrate a model like **Segment Anything (SAM)** or **YOLO-world** in ComfyUI to automatically segment the roof and feed the mask to the inpainting KSampler.
  2. **Option B (User-driven):** Provide a simple canvas/brush UI on the frontend so the user can paint over their roof to generate the mask, sending the mask coordinates to the backend.
  3. **Option C (Workflow-based):** Utilize an SDXL Inpainting model with a ControlNet Depth node to ensure that the layout structure of the roof is perfectly preserved while the texture and style are replaced.

### Gap 2: Supabase Integration
* **The Problem:** The project currently operates as a completely stateless local app. To build a premium visualizer that handles user profiles, stores generation history (so users can view past roofs), and loads an active admin-managed roof catalog, we need Supabase.
* **The Solution:** 
  1. Install `@supabase/supabase-js` and configure the environment variables.
  2. Set up Next.js middleware to protect the `/api/generate` route and main visualizer panel.
  3. Design the database schemas for `profiles`, `roof_tiles` (catalog), and `generations` (records of uploaded/generated image URLs).
  4. Connect Supabase Storage to upload original user photos and save the generated PNGs returned by ComfyUI.

### Gap 3: Comparison Slider
* **The Problem:** Users want a clean, interactive way to slide left-and-right to compare the "Before" (original) and "After" (generated) roofs. Currently, the UI only toggles them.
* **The Solution:** Build or import a lightweight CSS-based Before/After slider component (handling mouse/touch drag events to adjust the clip-path of the overlapping image).
