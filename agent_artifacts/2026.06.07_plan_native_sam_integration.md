# Implementation Plan - Migrate to Native SAM 3.1 Nodes (Finalized)

This plan outlines the changes needed to migrate our segmentation step from the third-party custom node extension (`comfyui_segment_anything`) to ComfyUI's **native SAM 3.1 implementation** (`SAM3_Detect`). This removes the need for custom node packs, simplifies the dependency tree, and resolves the `missing_node_type` warning for `SAMModelLoader`.

## Proposed Changes

### Next.js Backend API

#### [MODIFY] [route.ts](file:///c:/Users/geoff/github_repos/personal/roof_tile_visualizer/app/api/generate/route.ts)
Update the ComfyUI API workflow JSON to use native SAM 3.1 nodes instead of storyicon custom nodes:
1. Replace `SAMModelLoader` (Node 12) with a standard `CheckpointLoaderSimple` that loads `sam3.1_multiplex_fp16.safetensors` from the checkpoints directory.
2. Remove `GroundingDinoModelLoader` (Node 13).
3. Add a new `CLIPTextEncode` node (Node 18) dedicated to segmenting the `"roof"`. This reuses the `clip` output from the main SDXL model (Node 4).
4. Replace `GroundingDinoSAMSegment` (Node 14) with the native **`SAM3_Detect`** node. Connect it to the SAM 3.1 model (Node 12), the original image (Node 9), and the segmentation prompt conditioning (Node 18).
5. Update `VAEEncodeForInpaint` (Node 8) to use Node 14's native `MASK` output (index 0).

Here is the native SAM 3.1 connection layout:
- **Node 12 (`CheckpointLoaderSimple`)**: loads `sam3.1_multiplex_fp16.safetensors`.
- **Node 18 (`CLIPTextEncode`)**:
  - `text`: `"roof"`
  - `clip`: `["12", 1]` (uses CLIP model from SAM 3.1 checkpoint loader)
- **Node 14 (`SAM3_Detect`)**:
  - `model`: `["12", 0]`
  - `image`: `["9", 0]`
  - `conditioning`: `["18", 0]`
  - `threshold`: `0.3`
- **Node 8 (`VAEEncodeForInpaint`)**:
  - `mask`: `["14", 0]` (Native `SAM3_Detect` outputs the mask at index 0)
