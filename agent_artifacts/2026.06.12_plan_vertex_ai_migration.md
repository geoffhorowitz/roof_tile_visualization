# Vertex AI Migration Plan

Deploy the SDXL + ControlNet ComfyUI workflow to a Google Cloud Vertex AI custom prediction endpoint and integrate it into the Next.js backend (`app/api/generate/route.ts`).

## User Review Required

> [!IMPORTANT]
> **Authentication & Credentials**:
> In production (hosted on Google Cloud Run), the application will use the standard Cloud Run service identity (via the internal GCP metadata server) to authenticate to Vertex AI without requiring any hardcoded keys.
> For local development and testing, you will need to set the `VERTEX_SERVICE_ACCOUNT_KEY` environment variable in your `.env.local` to the JSON string of a Google Cloud Service Account key that has the **Vertex AI User** role.

## Open Questions

None. The integration format has been clarified:
- Vertex AI Endpoint receives a JSON payload: `{"instances": [{"workflow": workflow_json, "image": "base64_encoded_input_image"}]}`
- Endpoint returns a JSON response: `{"predictions": [{"image": "base64_encoded_output_image"}]}`
- The integration will fall back to local ComfyUI or mock mode if Vertex AI environment variables are not fully configured.

### Backend Logic

#### [NEW] [utils/vertexClient.ts](file:///home/gh-wsl/github_repos/personal/roof_tile_visualizer/utils/vertexClient.ts)
A standalone client to handle authenticated predictions on Vertex AI:
- Generates access tokens using local Service Account credentials (signing RS256 JWT using Node's standard `crypto` library) or requests them from the Cloud Run metadata server when running on GCP.
- Dispatches prediction calls to `https://${location}-aiplatform.googleapis.com/v1/projects/${projectId}/locations/${location}/endpoints/${endpointId}:predict`.

#### [MODIFY] [app/api/generate/route.ts](file:///home/gh-wsl/github_repos/personal/roof_tile_visualizer/app/api/generate/route.ts)
Modify the generation routing to check if Vertex AI is enabled and configured:
1. When enabled, read the file and convert it to a base64 string.
2. Construct the prediction payload containing the base64 image and the ComfyUI workflow JSON.
3. Call `predictVertexAI()` to get the generated base64 result.
4. Decode the result back to a Buffer and proceed with local output saving, Supabase storage upload, and database insertion, keeping all subsequent database and image logging behaviors intact.
