# Tile Catalog Thumbnails Integration

Add realistic thumbnail images for the roof tile selection catalog. This involves migrating 29 newly downloaded GAF and Owens Corning shingle images to the public assets directory, generating high-quality architectural tile thumbnails using AI, and refactoring the frontend catalog to present a premium, categorized user experience.

## User Review Required
> [!IMPORTANT]
> **Categorization and Filtering Strategy**: With 29 new shingle styles plus the 5 existing architectural styles, a flat list of 34 options would overwhelm the sidebar. I propose organizing the catalog into three tabs:
> 1. **Architectural Tiles** (Terracotta Red, Slate Grey, Mediterranean Clay, Forest Green Metal, Cedar Shake)
> 2. **GAF Timberline HDZ** (21 color options)
> 3. **Owens Corning Supreme** (8 color options)
>
> **Asset Generation**: Since the downloaded assets only cover shingle styles, I will generate custom photorealistic thumbnail assets for the 5 architectural styles (e.g. Terracotta, Slate, Green Metal, Cedar Shake) using `generate_image` to keep the visual design uniform and professional.

## Open Questions
No major open questions. We will use standard HTML image elements for simple and robust rendering of `.avif` and generated `.png` assets.

This directory will contain the user-provided architectural tile styles (filenames starting with `style_` e.g., `style_terracotta_red.png`) copied from `data/shingles/` to serve them statically to the client.

### Data Models & Catalog

#### [NEW] [config/tileCatalog.ts](file:///home/gh-wsl/github_repos/personal/roof_tile_visualizer/config/tileCatalog.ts)
A new catalog definition file that decouples the catalog definition from the `TileSelector.tsx` React component. It defines the categories, display names, hex codes, image paths, and CLIP generation prompts for all 34 styles.
