# Implementation Plan - SDXL Checkpoint Reversion (v2)

This plan outlines the changes needed to revert from the custom SDXL Inpainting checkpoint (`sd_xl_base_1.0_inpainting_0.1.safetensors`) back to the standard base SDXL checkpoint (`sd_xl_base_1.0.safetensors`) in the generation workflow. Testing has revealed that the custom inpainting model causes severe prompt color bias (such as rendering red terracotta tiles as dark brown/grey), whereas the base SDXL checkpoint adheres to color terms much more accurately.

## User Review Required
> [!IMPORTANT]
> The standard SDXL Base model will now be used for inpainting in ComfyUI. Users will need to have the `sd_xl_base_1.0.safetensors` file downloaded instead of or in addition to the inpainting model.

## Proposed Changes

### Next.js Backend API

#### [MODIFY] [route.ts](file:///home/gh-wsl/github_repos/personal/roof_tile_visualizer/app/api/generate/route.ts)

```json
      "4": {
        "class_type": "CheckpointLoaderSimple",
        "inputs": {
          "ckpt_name": "sd_xl_base_1.0.safetensors"
        }
      },
```
