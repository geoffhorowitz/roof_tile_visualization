# Execution Update: Color Prompt Debugging

**Date:** 2026-06-10

## Issue

Three renders showed incorrect colors:
- **Birchwood** (expected `#bfae9e` — light silver grey) → Rendered **dark forest green**
- **Golden Amber** (expected `#a67438` — warm golden-brown) → Rendered **flat olive/khaki**
- **Copper Canyon** (expected `#804a30` — reddish-brown) → Rendered approximately correctly but lacked shingle texture detail

## Root Cause Analysis

### Primary Issue: Ambiguous Brand Name Keyword at Start of Prompt

The SDXL model is a text-to-image model, and when given brand product names like `"birchwood, light grey asphalt shingles roof..."`, CLIP encodes the **first keyword** with the highest semantic weight. The model interprets brand names as natural scene descriptions:

| Brand Name | SDXL Semantic Interpretation | Result |
|---|---|---|
| **"Birchwood"** | Birch tree / birch wood forest | 🌲 Forest green |
| **"Golden Amber"** | Amber gemstone / amber liquid (yellowish, translucent) | 🫒 Flat yellow-olive |
| **"Chestnut Valley"** | Valley with chestnut trees | Potentially landscapes |
| **"Fox Hollow"** | Nature hollow / foxes | Potentially problematic |

### Secondary Issue: No Context Framing in Positive Prompt

The positive prompt was passed as-is (just the tile-specific prompt), with no context telling the model it's doing **inpainting on a house roof photo**. Without this framing, SDXL may hallucinate scene elements.

### Secondary Issue: Weak Negative Prompt

The negative prompt only excluded `"text, watermark, ugly, deformed, blurry"` — it said nothing about excluding natural environments, which allowed the "birch tree forest" interpretation to go through unchallenged.

## What Was Changed

### `config/tileCatalog.ts` — All 29 Tile Prompts Restructured

**Before (problematic pattern):**
```
'birchwood, light grey asphalt shingles roof, silver grey shingles'
```

**After (fixed pattern):**
```
'light silver grey asphalt shingle roof, pale warm grey shingles, soft taupe grey roof tiles, GAF Timberline birchwood color'
```

**Strategy:**
1. Lead with concrete, unambiguous color/texture descriptors (e.g., "light silver grey", "warm golden brown")
2. Include material context ("asphalt shingle roof")  
3. Add reinforcing color synonyms (e.g., "honey amber colored shingles, rich ochre gold brown")
4. Move brand name to the end as low-weight context (e.g., "GAF Timberline birchwood color")

### `app/api/generate/route.ts` — Workflow Prompt Nodes Updated

**Positive prompt — Before:**
```js
"text": prompt
```

**Positive prompt — After:**
```js
"text": `photorealistic house roof inpainting, ${prompt}, detailed shingle texture, high quality photography`
```

**Negative prompt — Before:**
```
"text, watermark, ugly, deformed, blurry"
```

**Negative prompt — After:**
```
"text, watermark, ugly, deformed, blurry, forest, trees, nature, landscape, grass, leaves, branches, birch trees, vegetation, outdoors scene"
```

## Expected Result

With these changes:
- **Birchwood** → Should now render as light silver-grey shingles
- **Golden Amber** → Should render as warm honey-amber/golden-brown shingles
- **All other tiles** → Improved color fidelity by leading with descriptive color terms
