# Execution Update: Debug Generation Pipeline

**Date:** 2026-06-07

## What was tried

1. **Baseline generation run (SAM prompt `"roof"`, ControlNet strength `0.8`, Denoise `0.85`)**:
   - Ran the pipeline with the original parameters on the house test image `data/house.jpg`.
   - **Result**: Confirmed the reported bug. Only the garage roof was selected and modified, and it was filled with a flat, textureless brown color.

2. **Prompt Tuning for SAM 3.1 (`"roof:10"`, ControlNet strength `0.8`, Denoise `0.85`)**:
   - Ran the pipeline with SAM text prompt set to `"roof:10"` to test if the tokenizer wrapper would enable multiple detections and merge them.
   - **Result**: Successfully selected the entire roof (both upper and lower portions). However, the generated roof remained flat and textureless because the ControlNet strength was too high.

3. **Combined Prompt Tuning and ControlNet strength reduction (`"roof:10"`, ControlNet strength `0.45`, Denoise `0.85`)**:
   - Ran the pipeline with the multiple detection SAM prompt and reduced the ControlNet Depth strength to `0.45` to allow the generator to add 3D barrel-style tile shapes.
   - **Result**: The entire roof was selected, but the texture remained a flat brownish-grey color. It did not render the rich red terracotta color.

4. **Combined Prompt Tuning and low ControlNet strength (`"roof:10"`, ControlNet strength `0.3`, Denoise `0.85`)**:
   - Ran the pipeline with the multiple detection SAM prompt and a very low ControlNet strength of `0.3`.
   - **Result**: The roof was selected, but some texture details bled outside the roof boundaries. The color still remained muted/flat.

5. **KSampler Denoise tuning to `1.0` (`"roof:10"`, ControlNet strength `0.45`, Denoise `1.0`)**:
   - Identified that `VAEEncodeForInpaint` zeroes out the masked region in the latent space. Retaining 15% of the original latent (at `denoise: 0.85`) meant the model was being forced to use a flat black/grey latent bias in the masked area, causing the output to be flat brownish-grey.
   - Set KSampler `denoise` to `1.0` to start with a clean slate of noise in the masked region.
   - **Result**: The entire roof was successfully selected and modified, rendering beautiful, highly detailed, photorealistic 3D terracotta red barrel tiles. The outline boundaries remained crisp and aligned.

---

## What worked

- **SAM Multi-detection Prompting**: Appending `:10` (i.e. `"roof:10"`) resolved the incomplete selection issue. By telling the native `SAM3_Detect` CLIP wrapper to allow up to 10 detections, all separate roof facets were successfully masked.
- **ControlNet Strength Optimization**: Setting the ControlNet Depth strength to `0.45` successfully resolved the shape constraint issues. It gave the model enough generative freedom to introduce 3D tile geometry while keeping the overall house perspective and roof bounds intact.
- **KSampler Denoise Optimization**: Setting the sampler `denoise` to `1.0` successfully resolved the flat grey/brown color issue. It allowed the inpainting model to generate the terracotta red tiles exactly as prompted instead of keeping the zero-masked latent bias.
- **Integration**: Programmatic changes in `app/api/generate/route.ts` and `tests/test_segmentation_api.py` were fully applied and tested. The schema validation passed with status SUCCESS.
