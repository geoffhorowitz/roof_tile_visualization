# Supabase Backend Integration

Integrate Supabase as the core backend for the Roof Tile Visualizer. This provides user authentication, database models for the roof tile catalog and user generations, and cloud storage for house images and generated results.

## User Review Required

> [!IMPORTANT]
> **Supabase Environment Variables**: The application will require a Supabase project. We need `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` to connect. We will provide a `.env.example` file.
> 
> **Development Fallback (Mock Mode)**: If these variables are not set in `.env.local`, the application will fall back to a "Mock Mode" that simulates successful logins, registers generation history in the browser's local state, and saves images locally. This ensures the application remains runnable and testable out-of-the-box.

> [!TIP]
> **Database Seeding**: We will provide a SQL script `supabase_schema.sql` to easily create the tables (`profiles`, `roof_tiles`, `generations`), triggers, and Storage buckets in the Supabase Dashboard SQL Editor, including pre-seeding the `roof_tiles` catalog from our existing `TILE_CATALOG`.
