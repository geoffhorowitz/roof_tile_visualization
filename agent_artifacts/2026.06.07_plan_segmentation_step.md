# Implementation Plan - Add Segmentation Step to ComfyUI Workflow (Finalized)

This plan outlines the changes needed to add a segmentation step to the ComfyUI workflow in the backend API. By isolating only the roof pixels, we prevent the model from modifying the entire image and ensure that only the roof is updated.

## User Comments Incorporated
- **SAM Model**: `sam3.1_multiplex_fp16.safetensors` (located in `comfyui/models/sams/`)
- **GroundingDINO Model**: `grounding-dino-tiny.safetensors` (located in `comfyui/models/grounding-dino/`)

## Proposed Changes

### Next.js Backend API

#### [MODIFY] [route.ts](file:///c:/Users/geoff/github_repos/personal/roof_tile_visualizer/app/api/generate/route.ts)
Update the ComfyUI API workflow JSON to:
1. Load the SAM model `sam3.1_multiplex_fp16.safetensors` using `SAMModelLoader`.
2. Load the GroundingDINO model `grounding-dino-tiny.safetensors` using `GroundingDinoModelLoader`.
3. Detect and segment the roof pixels using `GroundingDinoSAMSegment` with the prompt `"roof, roof tiles"`.
4. Replace the standard `VAEEncode` node with `VAEEncodeForInpaint` to encode only the masked roof region.
5. Pass the inpaint-encoded latent to the KSampler.
